# Convolution_of_4D_tensors
Сравнение прямой свёртки и метода im2col для 4D тензоров

## Документация

### Сборка

1. Скопируйте репозиторий
2. Перейдите в корневую папку проекта
3. Сгенерируйте систему сборки проекта, затем соберите проект коммандой в терминале (пример на Linux)

   `cmake -S . -B build -DCMAKE_BUILD_TYPE=Release && cmake --build ./build`

   command.
   
### Запуск программы

1. Всё в той же директории используйте комманду

   `build/main -i` 
 

 2. Всего доступны 4 режима запуска:
 
 
 * Режим тестирования свертки методом naive 

   `-n`,  `--naive`


 * Режим тестирования свертки методом im2col


   `-im`, `--im2col`


 * Режим интерактивного ввода, вывода данных


   `-i`,  `--interactive`


 * Вызов помощи


      `-h`,  `--help`    
3. При запуске программы 
* Введите резмерности тензора `input`: `N`,  `C`,  `H`,  `W`; 

* Соответсвтвующее число элементов `input`

* Размерность `K` `kernel`

* Соответствующее число элементов `kernel`


### Тестирование

1. В терминале используйте комманду

   `ctest --test-dir build/tests` 

2. Наблюдайте логи тестов


## Алгоритмы

### Naive

Алгоритм вычисляет свертку в лоб через 6 вложенных циклов. В каждой позиции окна $K \times K$ происходит скалярное умножение элементов входного тензора на веса ядра, и результат записывается в выходной тензор.

* Плюсы: Не требует дополнительной памяти.

* Минусы: Обладает крайне низкой пространственной локальностью данных (постоянные прыжки по памяти при считывании каналов и строк), что приводит к частым промахам кэша (cache misses) и невозможности эффективной векторизации.

### Im2col

Метод im2col решает проблему нерегулярного доступа к памяти, сводя операцию свертки к высокооптимизированному базовому матричному умножению (GEMM).

* Преобразование im2col: Входной тензор разворачивается в двумерную матрицу. Каждое локальное окно приема (receptive field) вытягивается в длинный столбец. В итоге формируется матрица размера ($C \times K \times K$) на ($H_{out} \times W_{out}$).

* Преобразование ядра: Фильтры ядра сплющиваются в матрицу размера $C$ на ($C \times K \times K$).

* GEMM: Выполняется стандартное матричное умножение этих двух матриц. За один проход вычисляются сразу все пространственные позиции для всех фильтров.

* Ограничения: В явном исполнении im2col дублирует данные в памяти (из-за перекрытия окон свертки), что приводит к значительному увеличению требуемой памяти. В статье упоминается, что современные аппаратные ускорители (такие как TPU) используют аппаратную модификацию implicit im2col (Channel-First layout), которая формирует эти матрицы на лету прямо в регистрах, тем самым избегая накладных расходов памяти. Представленный выше код реализует программную (explicit) версию, что является стандартом для таких бенчмарков.
